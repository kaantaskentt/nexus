# frontend/

Next.js 14 (app router) UI. Design system V2 (A16 "10k designer" bar) lives in `src/app/globals.css` + `tailwind.config.ts` (A15.1 — no ad-hoc colors): a warm surface ladder, a 4-step layered elevation scale (`--elev-1..4`), glass for floating chrome (`.glass` — top bar, drawer, sticky rails), a type scale, a density dial (`data-density`), and a motion vocabulary. Shared framer-motion variants (rise / stagger / drawer-spring) are in `src/lib/variants.ts` so motion is consistent, not per-component; `prefers-reduced-motion` is honored globally. Trust rules are enforced in `src/components/` (confidence badges, paraphrased employee evidence, facts-only why-lines). Entry: `npm run dev`; screens under `src/app/w/[slug]/` (admin) and `src/app/i/[token]/` (respondent); data is live via `src/lib/live.ts` against the API at `:8000` — conversation paths never mock (`api()` reads are `cache: "no-store"` so a fresh compile or edit never renders stale).

**Auth (A17):** `src/middleware.ts` gates `/` and `/w/*` behind a Supabase session (`@supabase/ssr`); `/login` is the only public admin page and `/i/[token]` interview links stay open by design. No signup — admins are created manually (`backend/scripts/create_admin.py`). Needs `NEXT_PUBLIC_SUPABASE_URL` + `NEXT_PUBLIC_SUPABASE_ANON_KEY` (in `.env.local`, and on Vercel). **New-company flow:** the picker doubles as a workspace switcher with "Add company"; a fresh tenant lands on a guided empty state (`components/snapshot/DiscoveryUpload.tsx`) to paste/drop the CEO call and watch the snapshot compile progressively, with an optional website scan.
